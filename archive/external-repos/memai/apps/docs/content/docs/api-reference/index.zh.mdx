# API 参考

这是 Memai 的完整 API 参考文档。所有端点都是 RESTful 的，返回 JSON 响应。

## 基础 URL

```
https://api.memai.dev
```

## 身份验证

所有请求都需要在 Authorization 头中包含 API 密钥：

```
Authorization: Bearer mm_your_api_key
```

详见 [身份验证](/zh/docs/getting-started/authentication)。

## API 版本

当前 API 版本是 **v1**。所有端点都以 `/v1/` 为前缀。

## 端点概览

### 记忆

| 方法 | 端点 | 描述 |
|------|------|------|
| `POST` | `/v1/memories` | 创建新记忆 |
| `POST` | `/v1/memories/search` | 语义搜索记忆 |
| `GET` | `/v1/memories` | 列出记忆 |
| `GET` | `/v1/memories/:id` | 获取特定记忆 |
| `DELETE` | `/v1/memories/:id` | 删除记忆 |

### 实体

| 方法 | 端点 | 描述 |
|------|------|------|
| `POST` | `/v1/entities` | 创建新实体 |
| `GET` | `/v1/entities` | 列出实体 |
| `DELETE` | `/v1/entities/:id` | 删除实体 |

### 关系

| 方法 | 端点 | 描述 |
|------|------|------|
| `POST` | `/v1/relations` | 创建新关系 |
| `GET` | `/v1/relations` | 列出关系 |
| `DELETE` | `/v1/relations/:id` | 删除关系 |

## 常见响应码

| 状态码 | 描述 |
|--------|------|
| `200` | 成功 |
| `201` | 已创建 |
| `400` | 请求错误 - 参数无效 |
| `401` | 未授权 - API 密钥无效或缺失 |
| `403` | 禁止访问 - 配额超限 |
| `404` | 未找到 |
| `429` | 请求过多 - 速率限制 |
| `500` | 服务器内部错误 |

## 错误响应格式

所有错误遵循以下格式：

```json
{
  "error": {
    "code": "ERROR_CODE",
    "message": "人类可读的错误信息"
  }
}
```

## 分页

列表端点支持以下分页参数：

| 参数 | 类型 | 默认值 | 描述 |
|------|------|--------|------|
| `page` | number | 1 | 页码 |
| `limit` | number | 20 | 每页数量（最大 100） |

响应包含分页信息：

```json
{
  "items": [...],
  "pagination": {
    "page": 1,
    "limit": 20,
    "total": 150,
    "totalPages": 8
  }
}
```
