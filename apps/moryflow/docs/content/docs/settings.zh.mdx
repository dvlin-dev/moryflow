# 设置

MoryFlow 的设置现在按职责分区，不再是一长串混在一起的开关。

## 当前设置分区

当前桌面端设置固定分成这些 section：

1. **Account**
2. **General**
3. **Personalization**
4. **Providers**
5. **MCP**
6. **Cloud Sync**
7. **About**

## 每个分区负责什么

### Account

这里负责身份和计费相关能力：

- 登录或注册
- 管理个人资料
- 查看会员状态
- 购买积分
- 删除账号

[查看账号与会员 →](/zh/docs/settings/account-and-membership)

### General

这里负责桌面端的通用偏好和更新状态：

- 当前更新状态
- 主题相关默认项
- 以及其他通用应用行为

[了解下载与更新 →](/zh/docs/getting-started/updates)

### Personalization

这里负责你的个性化指令和工作风格偏好。

当前最核心的是自定义指令，它和模型、服务商配置是分开的。

### Providers

这里负责决定 Mory 可以使用哪些模型服务商：

- 内置服务商
- 自定义服务商
- API Key
- 启用的模型
- 默认模型选择

[查看 Providers 与 Models →](/zh/docs/settings/providers-and-models)

### MCP

这里负责连接外部 MCP Server：

- 添加 stdio 或 HTTP server
- 使用 preset
- 测试连通性
- 启用或停用 server

[查看 MCP 设置 →](/zh/docs/settings/mcp)

### Cloud Sync

这里负责当前 vault 的内置云同步：

- 是否已登录
- vault 绑定
- 同步开关
- 当前同步状态
- 冲突副本处理

[查看 Cloud Sync 设置 →](/zh/docs/settings/cloud-sync)

### About

这里主要放应用信息和版本细节。

## 设置存在哪里

MoryFlow 的设置保存在本地设备上。你的笔记 vault 和设置并不是同一层东西。

- vault 保存 Markdown 内容
- 设置保存应用配置、服务商信息和本地偏好

<Callout title="一个好理解的方法">
  把 vault 当成内容层，把 Settings 当成运行层。
</Callout>
