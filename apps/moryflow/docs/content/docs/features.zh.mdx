# 功能概览

现在的 MoryFlow 已经不只是“笔记 + 聊天框”。当前产品把 local-first vault、写作界面、AI 工作区和可选的外部能力放在了一起。

## 当前产品的四个面

### Vault

你的 vault 是本地内容层。

- 笔记都是普通 Markdown 文件
- 你始终直接掌控自己的文件
- 离开 MoryFlow，内容本身仍然可用

[了解更多 →](/zh/docs/features/vault)

### Editor

编辑器负责阅读、写作和整理内容。

- 富文本编辑
- slash 风格插入流程
- 多笔记工作流
- 和右侧 Mory 工作区紧密配合

[了解更多 →](/zh/docs/features/editor)

### AI Assistant

Mory 是工作区里的动作层。

- 可以从空白输入开始，也可以从 pre-thread 引导入口开始
- 可以引用文件和选区
- 可以在任务进度可见的情况下逐步完成工作
- 可以把模型、内置能力和 MCP 一起用起来

[了解更多 →](/zh/docs/features/ai-assistant)

### MCP

MCP 负责把 Mory 的能力扩展到内置环境之外。

- 连接外部 server
- 添加专项能力
- 把运行时扩展和模型服务商配置分开管理

[了解更多 →](/zh/docs/features/mcp)

## 这些部分怎么配合

现在更常见的工作流是：

1. 把源材料放进 vault
2. 在 editor 里写和整理
3. 让 Mory 帮你分析、改写、规划或发布
4. 只有在需要外部能力时，再接入 MCP

## 和旧文档相比，哪里变了

旧版 MoryFlow 文档常把产品讲成：

- 笔记文件夹
- 编辑器
- Mory
- 一份固定工具清单

这已经不足以解释当前产品。现在更关键的是 Chat 入口、任务进度、Providers 配置和云端能力边界。
