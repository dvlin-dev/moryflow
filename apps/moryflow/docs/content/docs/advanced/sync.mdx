# Multi-device Sync

MoryFlow now has two different sync stories, and they should not be mixed together:

1. **Built-in Cloud Sync**
2. **External folder sync**

## Option 1: Built-in Cloud Sync

Built-in Cloud Sync is the app-managed sync path for your current vault.

Use it when you want MoryFlow itself to manage:

- vault binding
- sync on/off
- sync status
- retry or recovery state
- conflict-copy handling

### What It Requires

Built-in Cloud Sync currently depends on:

1. Being signed in
2. Having a vault open

If one of those is missing, the app stops at that state first instead of pretending sync is ready.

### What You Can See

Inside Cloud Sync settings, the app can show:

- whether the current vault is bound
- whether sync is enabled
- current engine state
- latest sync time
- advanced usage details
- recovery or conflict prompts when needed

### When to Prefer It

Prefer built-in Cloud Sync when you want the app to be aware of sync state directly.

That is the current MoryFlow-native path.

## Option 2: External Folder Sync

External folder sync means you keep using your local vault as normal, but place that folder inside another sync system outside MoryFlow.

This is still valid if you specifically want a generic file-sync layer outside MoryFlow.

## How to Choose

### Choose Built-in Cloud Sync if:

- you want sync status inside the app
- you want MoryFlow to manage the current vault binding
- you want app-level prompts for recovery and conflicts

### Choose External Folder Sync if:

- you already have a strong file-sync workflow outside MoryFlow
- you want a generic folder-sync layer and do not need app-managed sync state
- you are intentionally treating MoryFlow as one client among many editors

## Conflict Handling

Whichever path you use, conflicts can still happen if the same file changes in more than one place at the same time.

Good practice:

1. Finish work on one device before switching to another
2. Wait for sync to settle before editing again
3. Treat conflict copies as review items, not as files to ignore

## Important Boundary

Do not think of external folder sync as "the same thing but cheaper."

Built-in Cloud Sync and external folder sync solve different problems:

- one is app-managed
- one is file-system-managed
