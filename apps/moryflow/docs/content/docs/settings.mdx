# Settings

MoryFlow's settings are organized by responsibility, not by one long list of unrelated toggles.

## Current Settings Sections

The current desktop app settings are organized into these sections:

1. **Account**
2. **General**
3. **Personalization**
4. **Providers**
5. **MCP**
6. **Cloud Sync**
7. **About**

## What Each Section Is For

### Account

Use this section for identity and billing:

- Sign in or register
- Manage your profile
- Check your membership
- Buy credits
- Delete your account

[Open Account & Membership →](/docs/settings/account-and-membership)

### General

Use this section for desktop-level preferences and update status:

- Current update state
- Theme-related defaults
- Other general app behavior as it becomes available

[Learn about Download & Updates →](/docs/getting-started/updates)

### Personalization

Use this section for your own instructions and working style preferences.

Right now, this is where custom instructions live. It is separate from provider and model setup.

### Providers

Use this section to decide which model providers Mory can use.

- Built-in providers
- Custom providers
- API keys
- Enabled models
- Default model selection

[Open Providers & Models →](/docs/settings/providers-and-models)

### MCP

Use this section to connect external MCP servers.

- Add stdio or HTTP servers
- Use presets
- Test connectivity
- Enable or disable servers

[Open MCP Settings →](/docs/settings/mcp)

### Cloud Sync

Use this section to manage MoryFlow's built-in cloud sync for your current vault.

- Sign-in requirement
- Vault binding
- Sync on/off
- Current sync status
- Conflict copy handling

[Open Cloud Sync Settings →](/docs/settings/cloud-sync)

### About

Use this section for app information and version details.

## Where Settings Are Stored

MoryFlow settings are stored locally on your device. Your notes vault and your settings are not the same thing.

- The vault stores your Markdown files
- Settings store app configuration, provider setup, and local preferences

<Callout title="Good default">
  Treat your vault as your content layer, and treat Settings as your runtime layer.
</Callout>
