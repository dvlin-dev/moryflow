# Mory

Mory 是 MoryFlow 里的 AI 工作区，不只是一个聊天框。这里会把笔记、文件、模型、运行时能力和任务进度放到同一个工作流里。

## 现在怎么开始一次对话

当前在 MoryFlow 里，最常见的两种入口是：

### 直接从会话开始

如果你已经知道自己要什么，可以直接在输入框里打出来并发送。

### 从 Pre-thread 入口开始

如果你还没想好怎么开头，pre-thread 区域会帮你从这些方式开始：

- 引导式 starter task
- 推荐 skills
- 先预填 prompt，而不是直接替你发送

也就是说，应用会帮你起步，但最终 prompt 仍然由你确认。

## Mory 会处理哪些上下文

在一次工作流里，Mory 当前可以结合多种上下文：

- 当前笔记
- vault 里的其他文件
- 选中的文本
- 上传的附件
- 你当前选的模型
- 已连接的 MCP 能力

## 透明的任务进度

当一件事需要多步完成时，MoryFlow 会在聊天区域里显示任务进度。

这个面板是刻意做轻的：

- 没有一套独立的后台任务产品心智
- 也不是另一个 task app
- 它只是把当前会话正在做什么清楚地展示出来

## 模型和访问方式

Mory 不是只有一种固定智能来源。

它的行为取决于：

- 你配置了哪家 provider
- 当前选的是哪个 model
- 有没有打开额外的 MCP 能力
- 你给了它哪些文件或选区作为上下文

所以模型配置属于 **Settings**，而不是内容文件本身。

## 可以让 Mory 做什么

典型请求包括：

- 总结笔记
- 把零散想法整理成计划
- 改写文章
- 对比多个文件里的观点
- 生成可发布页面
- 在有 web access 或 MCP 的情况下补充外部信息
