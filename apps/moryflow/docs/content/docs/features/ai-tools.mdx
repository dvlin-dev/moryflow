# Mory Runtime Capabilities

This page is intentionally high level.

Older docs described Mory with a fixed built-in tool list. That no longer ages well enough to serve as a stable user-facing contract.

## A Better Way to Think About It

When Mory works on something, it may use capability categories such as:

- file access inside your current workspace
- note creation or editing
- search across your local material
- web access when available
- command execution in supported contexts
- external capabilities provided through MCP

The exact runtime path depends on the current app version, your provider setup, your permissions, and any connected MCP servers.

## What Stays Stable

The stable part is not the exact tool inventory. The stable part is the workflow:

1. You give Mory a request
2. You provide context when needed
3. Mory shows what it is doing
4. Multi-step work appears in task progress
