# Cloud Sync

**Cloud Sync** 分区负责当前 vault 的内置同步。

它和给本地文件夹单独配一套外部同步客户端不是同一件事。

## 使用前提

当前 Cloud Sync 依赖两件事：

1. 你已经登录
2. 你已经打开了一个 vault

如果缺任何一个，设置页会先停在对应状态，不会直接显示完整同步控制。

## 当前页面状态

Cloud Sync 现在有 4 个主要状态：

1. `auth-loading`
2. `unauthenticated`
3. `missing-vault`
4. `ready`

也就是说，这一页不是永远长得一样，它会先判断当前应用状态下能不能真正开始同步。

## 在 Ready 状态下可以做什么

进入 ready 后，你可以：

- 绑定当前 vault
- 打开或关闭同步
- 查看当前同步状态
- 看最近一次同步时间
- 再次触发同步
- 展开高级用量信息

## 冲突副本与恢复

如果同步状态需要额外处理，Cloud Sync 还会显示这些提示：

- 需要恢复
- 离线重试
- 初始设置未完成
- 已生成冲突副本

如果真的生成了冲突副本，应用可以直接把你带到第一个冲突文件。

## 内置 Cloud Sync 和外部文件夹同步的区别

如果你想让 MoryFlow 在应用内管理当前 vault 的同步状态，用内置 Cloud Sync。

如果你只是想在 MoryFlow 外面做一层通用文件同步，用外部文件夹同步。

两者有关联，但不是同一个功能。
