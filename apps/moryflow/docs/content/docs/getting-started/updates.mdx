# Download & Updates

MoryFlow separates **manual downloads** from **in-app update delivery**.

## Public Download Entrypoints

- **GitHub Releases** is the official page for manual downloads and release notes
- **download.moryflow.com** is reserved for the app's update feed

If you want to download a build manually, read release notes, or find an older version, always start from GitHub Releases.

## Current Public Status

- Public downloads are currently available for:
  - macOS (Apple Silicon)
  - macOS (Intel)
- Windows is not currently published
- The public website may surface beta builds before they become the default stable path

## How In-App Updates Work

MoryFlow follows this update flow:

1. The app checks for updates automatically
2. You decide when to **download**
3. You decide when to **restart and install**

That means update **checks** are automatic, but **download** and **install** stay under user control.

## Stable and Beta

MoryFlow supports two update channels:

- **Stable**: the default long-term path
- **Beta**: earlier builds for users who want to validate newer changes sooner

The public website and docs may point to a beta release when beta is the current public distribution path.

## Where to Change Update Settings

Inside the app:

1. Open **Settings**
2. Go to **General**
3. Check update status, available download action, and current channel there

If the UI wording changes later, the product still keeps the same split between automatic checks and user-confirmed install.

## If an Update Fails

Use this fallback order:

1. Open [GitHub Releases](https://github.com/dvlin-dev/moryflow/releases)
2. Download the matching installer manually
3. Install it over your existing app
