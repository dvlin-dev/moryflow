## 三种产品哲学

```
AutoClaw / QClaw ：  IM 入口 → AI 执行 → 结果回流       （执行型 Agent）
Moryflow         ：  知识库  → AI 思考  → 内容发布        （知识型 Agent）
海外生态产品      ：  OpenClaw → 包装/部署/垂直化 → 商业化  （服务型/平台型）
```

AutoClaw 和 QClaw 是"执行型 Agent"——帮用户干活，但不帮用户积累知识。它们没有编辑器、没有知识库、没有发布能力。

Moryflow 的差异化在于"知识型 Agent"——AI 长在知识库上，基于用户积累的笔记和素材做深度工作，最终产出可发布的内容。

## 定位建议

### 核心叙事转变

从工具视角转向结果视角：

| 当前口径（工具视角）   | 建议口径（结果视角）        |
| ---------------------- | --------------------------- |
| AI 笔记工具            | 想法变成网站，一步到位      |
| 支持 20+ AI 模型       | 用你自己的 AI，数据不出电脑 |
| 17 个内置 Skills       | AI 帮你写、帮你做、帮你发   |
| Markdown 编辑器 + 发布 | 写完即发布，你的内容你做主  |

**一句话定位**：Moryflow — 从想法到发布，你的 AI 内容工作台。

### 不跟进的方向

| 方向                            | 原因                                          |
| ------------------------------- | --------------------------------------------- |
| 抢 IM 入口（微信/飞书为主入口） | 大厂生态级优势，IM 集成作为补充能力而非主战场 |
| 拼 Skills 数量                  | 数量不是壁垒，做少而精 + 场景化组合           |
| 做通用 AI Agent                 | AutoClaw/QClaw 已占位，Moryflow 无优势        |

## 优先级路线图

### P0 — 结果叙事（口径调整）

成本：低。纯认知调整，改官网文案和产品内引导。

不再说"我有什么功能"，而是说"你能得到什么结果"。竞争维度从"执行效率"转向"内容产出"——帮用户从零到一把内容发出去。

### P1 — 发布为核心身份（产品调整）

成本：中。需要调整产品优先级和 UI 信息层级。

发布是 Moryflow 最独特的能力——AutoClaw 没有、QClaw 没有、Obsidian 没有、Notion 的发布很弱。当前"Sites"只是侧边栏一个 Tab，应提升为核心体验：

- 打开 Moryflow，首先感知到"你已经发布了 N 篇内容"
- 发布是主故事线，AI 和编辑器是达成发布结果的手段
- 不是"Moryflow 也可以发布网站"，而是"Moryflow 就是让你发布的"

### P2 — 人群锚定（模板 + 套装）

成本：中。需要设计和制作预置内容。

明确 2-3 个典型用户画像，为每个人群做预置 Vault 模板 + Skills 组合：

| 人群             | 痛点                       | 解决方案                              |
| ---------------- | -------------------------- | ------------------------------------- |
| 独立创作者/博主  | 想持续输出但没技术能力建站 | 写笔记 → AI 润色 → 一键发布           |
| 知识型自由职业者 | 想把专业知识变成内容产品   | Vault 积累 → AI 编排 → 发布为专业网站 |
| 小团队内容运营   | 多人写内容、统一发布       | 本地知识库 + AI 辅助 + 品牌站点       |

### P3 — 价值观包装（品牌心智）

成本：低。品牌和传播层面的调整。

把"本地 + BYOK"从技术参数包装成价值主张：

| 技术语言             | 价值观语言                                   |
| -------------------- | -------------------------------------------- |
| 支持 20+ AI Provider | 你的 AI 你选，你的数据你管                   |
| 本地 Vault 架构      | 你的知识留在你手里，不是在某个公司的服务器上 |
| BYOK 多模型          | 不绑定任何一家 AI，永远用最好的那个          |

与大厂产品形成意识形态对立：飞书/钉钉/微信 = 数据在它们的云上；Moryflow = 数据在你的电脑上。
