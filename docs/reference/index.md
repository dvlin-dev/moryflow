# Reference

本目录承接“协作参考、工程规范、验证流程、部署/构建基线”这类按需查阅内容。

约束：

1. `docs/reference` 不承载产品架构与协议事实；这类内容仍归 `docs/design/*`。
2. `docs/reference` 只保留稳定规则、操作基线与跨模块共用规范。
3. `docs/reference` 采用扁平结构，禁止继续创建子目录。
4. 仅在规则、入口或基线失真时更新；禁止写时间线、PR 过程与验证播报。
